## 1. 快速開始

### 1.1 啟動應用程式

```bash
cd task-time-tracker
npm run dev
```

瀏覽器開啟 `http://localhost:5173` 即可使用。所有資料存在瀏覽器的 `localStorage` 中，無需後端伺服器。

> ⚠ **重要**：資料存在瀏覽器本機，清除瀏覽器資料（如清除 Cookie/快取）會導致遺失。請定期至「系統設定」備份。

### 1.2 首次使用建議流程

1. **前往「系統設定」** → 設定任務分類（如：開發、會議、管理）和時間分類（如：程式開發、議題討論、閱讀學習）
2. **前往「任務管理」** → 建立你的第一批任務，設定 WBS 階層關係
3. **回到「排程視圖」** → 在時間軸上記錄你每天的工作時間
4. **查看「統計報表」** → 觀察你的工時分配
5. **使用「週報生成」** → 產出 PlantUML 圖表與進度追蹤表

### 1.3 核心概念

在深入使用前，理解以下三個概念可以避免很多困惑：

#### 任務（Task）vs 時間紀錄（Timeslot）

這是兩個獨立的東西，但會連結在一起：

- **任務（Task）**：你要做的「工作包」，描述 _要做什麼_。例如：「實作使用者登入功能」。任務本身不記錄時間。
- **時間紀錄（Timeslot）**：你實際花費的時間片段，記錄 _何時做_。例如：「今天 9:00–11:30 在做登入功能」。Timeslot 可以選擇性地關聯到某個任務。

兩者分離的好處：同一個任務你可以跨多天多次記錄工時；也可以建立不關聯任何任務的 Timeslot（例如臨時協助別人），時間仍計入統計。

#### 任務分類 vs 時間分類

這兩個分類系統各有不同用途，常讓初次使用者混淆：

| | 任務分類 | 時間分類 |
|--|---------|---------|
| **屬於** | Task（任務） | Timeslot（時間紀錄） |
| **代表** | 這個工作屬於哪個領域 | 這段時間在做哪種活動 |
| **範例** | 開發專案、客戶服務、行政管理 | 程式開發、議題討論、閱讀學習、固定會議 |
| **統計用途** | 讓你看出各專案/領域佔用多少工時 | 讓你看出各種活動類型佔用多少工時 |

**舉例**：你在做「電商平台」專案（任務分類：電商專案）的「購物車 API」，今天花了 3 小時在寫程式（時間分類：程式開發）、1 小時在討論規格（時間分類：議題討論）。兩個分類分別回答「為哪個客戶做事」和「在做什麼類型的工作」。

#### 工作產出（WorkOutput）是什麼

工作產出是任務的「可見成果」，用來佐證你的工作結果。例如一個「撰寫技術文件」任務，它的產出可以是：
- 一份 API 文件（有形產出，附上連結）
- 一個設計決策（無形產出，附上說明）

工作產出在週報進度表中會以子列呈現，讓報告有具體憑據。週期型任務（如每週例會）的產出可以設定「歸屬期間」，讓不同週的產出可以在各週的報告中分別顯示。

## 2. 全域功能

### 2.1 導覽列

頁面頂部的 AppBar 提供所有頁面的快速導覽：

| 按鈕 | 頁面 | 路徑 |
|------|------|------|
| 任務管理 | WBS 任務清單 | `/tasks` |
| 工作產出 | 產出追蹤 | `/outputs` |
| 排程視圖 | Outlook 風格時間軸 | `/` |
| 統計報表 | 圓餅圖工時統計 | `/reports` |
| 週報生成 | PlantUML 圖表＋進度表 | `/weekly` |
| 封存庫 | 封存任務管理 | `/archive` |
| 系統設定 | 分類/假日/備份 | `/settings` |

### 2.2 復原（Undo）

導覽列右側的 ↩ 按鈕可復原最近的操作（**最多 20 步**）。支援復原的操作包括：新增/編輯/刪除任務、新增/編輯/刪除時間紀錄、封存/還原任務、調整任務排序等。

### 2.3 深色模式

導覽列右側的 🌙/☀ 按鈕切換深色/亮色主題。設定會自動儲存，下次開啟時維持上次的選擇。

## 3. 排程視圖（Dashboard）

排程視圖是你日常記錄工作時間的主要介面，採用 Outlook 行事曆風格，以直覺的視覺化方式呈現每天的時間分配。

### 3.1 檢視控制

頁面上方提供以下控制項：

| 控制項 | 說明 |
|--------|------|
| **日期選擇器** | 點擊選擇要檢視的日期 |
| **← → 箭頭** | 快速切換前/後一天（日模式）或一週（週模式） |
| **檢視模式** | `日`（單天）、`週5`（週一～五）、`週7`（週日～六） |
| **縮放密度** | `精簡`（每小時 60px）、`詳細`（每小時 120px） |
| **色塊顏色** | 以「任務分類」或「時間分類」區分色塊顏色 |
| **顯示說明** | 開關 Timeslot 的滑鼠懸停備註提示 |

> **提示**：色塊顏色選「任務分類」時，你可以一眼看出今天花在哪個專案最多；選「時間分類」則可以看出你今天在做哪類活動（開發/會議/學習等）。兩種視角各有用途。

### 3.2 時間軸

- 顯示 0:00 到 23:00 的完整 24 小時，開啟時自動捲動到早上 7:00
- 今天的日期欄位有藍色背景高亮
- 紅色水平線標示目前時刻
- 已記錄的 Timeslot 以色塊呈現，顯示任務名稱（優先顯示別名）與時間範圍

### 3.3 新增時間紀錄

**點擊時間軸空白處** → 彈出「快速新增時間紀錄」對話框：

- **任務**（選填）：從未封存的任務清單中選取，也可不選（未分類）
- **時間分類**（選填）：如程式開發、固定會議等
- **日期**：自動帶入你點擊的那一天
- **開始/結束時間**：自動對齊到 15 分鐘，結束時間預設加 30 分鐘
- **說明**（選填）：備註文字
- **「建立新任務」捷徑**：若想記錄的任務尚未建立，可直接從此開新任務，完成後自動回到新增畫面並預選該任務

> **不一定要先有任務才能記錄時間**。如果是臨時協助同事、突發的技術問題排查，可以直接建立 Timeslot 只填時間分類與說明，不關聯任何任務。這段時間仍會計入統計。

### 3.4 編輯與刪除時間紀錄

**單擊** Timeslot 色塊 → 開啟「編輯時間紀錄」對話框，可修改所有欄位或刪除該筆紀錄。

### 3.5 從 Timeslot 快速編輯任務

**雙擊** Timeslot 色塊 → 直接開啟該任務的 TaskForm 編輯對話框。

> 系統以 250ms 延遲區分單擊（編輯時間紀錄）與雙擊（編輯任務）。如果你發現雙擊沒反應，可能是點擊間隔太長，再試一次即可。

## 4. 任務管理

### 4.1 WBS 階層結構

任務支援最多 **5 層**的父子階層關係，以 WBS 編號呈現（如 1、1.1、1.1.2）。子任務會以縮排方式顯示在父任務下方。

> **WBS（Work Breakdown Structure）** 是一種將大型工作拆解成可管理小單元的方式。例如「電商平台開發」（第 1 層）→「購物車功能」（1.1）→「API 設計」（1.1.1）。系統自動計算並顯示 WBS 編號。

### 4.2 篩選與搜尋

頁面上方提供多維度篩選：

- **搜尋欄**：全文搜尋任務名稱、別名、說明
- **狀態篩選**：多選，可同時篩選多個狀態
- **分類篩選**：多選，依任務分類篩選
- **標籤篩選**：多選，依自訂標籤篩選
- **日期範圍**：依預估開始/完成日期篩選
- **縮合/展開**：一鍵折疊或展開所有有子任務的任務列

### 4.3 任務狀態

| 狀態 | 說明 | 可封存 |
|------|------|--------|
| 待規劃 (BACKLOG) | 想法或計畫，尚未決定何時執行 | 否 |
| 待處理 (TODO) | 已確定要做，正在排入工作計畫 | 否 |
| 進行中 (IN_PROGRESS) | 正在執行 | 否 |
| 已暫停 (PAUSED) | 暫時停止，**必須填寫原因** | 否 |
| 已完成 (DONE) | 已完成 | 是 |
| 已取消 (CANCELLED) | 決定不做 | 是 |

> **BACKLOG vs TODO 的差異**：BACKLOG 是「我知道有這件事，但還沒決定什麼時候做」；TODO 是「已確定要做，但還沒開始」。兩者的使用邊界可以依個人習慣調整，不必過於糾結。

> **何時封存、何時刪除？** 已完成或取消的任務建議「封存」而非「刪除」——封存後任務的 Timeslot 仍計入統計，你還可以在封存庫中查閱歷史；刪除則會一併刪除子任務，Undo 也只有 20 步的保護。**原則：有完整工時紀錄的任務優先封存，確認不需要保留記錄的才刪除。**

### 4.4 任務表格

| 欄位 | 說明 |
|------|------|
| No. | WBS 編號，可點擊折疊/展開子任務 |
| 任務名稱 | 標題、別名（灰色小字）、標籤（Chip） |
| 任務分類 | 所屬分類 |
| 預估日期 | 預估開始～完成日期，逾期任務會標紅色 |
| 實際日期 | 從 Timeslot 自動計算（最早開始/最晚結束） |
| 狀態 | 狀態 Chip ＋ 完成度百分比 |
| 累計工時 | 所有關聯 Timeslot 的工時加總 |
| 操作 | 排序 ↑↓、層級調整 ←→、編輯、建子任務、封存、刪除 |

### 4.5 建立/編輯任務（TaskForm）

TaskForm 對話框包含以下欄位群組：

**基本資訊**
- 任務名稱（必填）
- 別名（最多 10 字）：在排程視圖色塊上優先顯示，適合用來縮短長任務名稱
- 上層任務（WBS 歸類）：選擇父任務或設為頂層；系統自動過濾掉自身及後代以防止循環參照
- 任務詳細說明
- 標籤（最多 3 個）：可用來標記優先度（高/中/低）、性質（跨部門/外部依賴）等

**分類與狀態**
- 任務分類（從系統設定中選取）
- 任務狀態（六種狀態，暫停時須填原因）

**時程**
- 預估開始/完成日期（若勾選顯示於甘特圖則為必填）
- 實際日期（唯讀）：自動由關聯的 Timeslot 推算，不需手動填寫。實際開始 = 最早一筆 Timeslot 的開始時間；實際完成 = 最晚一筆 Timeslot 的結束時間。

**人員**
- 任務負責人
- 任務指派人

**追蹤設定**：見下一節詳細說明。

**完成度歷史快照**（僅編輯模式＋追蹤完成度時可見）

這個區塊用來補填或修改**過去各期**的完成度數值，解決這個問題：「如果我上週忘記更新完成度，這週的週報前期欄位會顯示『—』，讓進度表看起來沒有基準可比較。」

- 展開後顯示各週快照清單（可新增、編輯、刪除）
- 日期自動對齊至該週週日（系統以週日為一週起始）
- 有快照時底部自動顯示**折線圖**（任務整體 + 各工作產出各一條線）

> 正常使用時，每次修改「整體完成度 %」欄位並儲存，系統會自動建立當週快照，不需手動操作這個區塊。只有在需要補填過去資料時才需要展開它。

**工作產出**
- 可新增多筆產出，每筆設定名稱、類型（有形/無形）、完成度
- 有形產出填寫「相關連結」（URL 或路徑）；無形產出填寫「說明/摘要」
- **歸屬期間**（週期型產出選填）：如果這筆產出屬於某個特定週/月（如「第 10 週會議紀錄」），填入該期間內任意日期；若是一份持續更新的長期文件（如「系統架構文件」），則留空
- 每筆產出各自有完成度快照管理
- **複製為下期**（📋 按鈕）：點擊後自動複製本筆產出，歸屬期間自動 +7 天，完成度歸零，適合週期型任務每週新增產出

### 4.5.1 四個追蹤選項判斷指南

任務的四個核取方塊控制不同的報告與追蹤行為，預設全部勾選。這份指南幫助你判斷何時應該取消勾選。

---

#### 追蹤完成度 %

**核心問題：這個任務有沒有「完成」這個概念？**

| 情況 | 建議 |
|------|------|
| 任務有明確的終點（開發完一個功能、寫完一份報告） | ✅ 勾選 |
| 任務是週期性持續進行的，永遠不會「完成」 | ❌ 不勾選 |

**為什麼重要？** 勾選後，每次修改完成度會自動建立週快照，週報進度表才能比較「前期 vs 本期」的進展差值。若在一個沒有終點的任務上強行追蹤百分比，你每週都不知道要填多少，而且系統很可能把它放進「無進展」表。

若不勾選，系統改用「本期是否有 Timeslot 活動或新的工作產出」來判斷有無進展，更符合這類任務的實際情況。

**適合取消的例子**：每週例會、On-call 維運支援、人員招募、長期客戶關係維護

---

#### 顯示於 WBS

**核心問題：這個任務是一個可識別的工作包，值得出現在工作分解結構中嗎？**

| 情況 | 建議 |
|------|------|
| 任務是一個有意義的工作項目或里程碑 | ✅ 勾選 |
| 任務是行政雜務、過度細碎的步驟，或在 WBS 圖中會造成干擾 | ❌ 不勾選 |

**為什麼重要？** WBS 圖是讓別人（或未來的自己）快速理解你在做什麼的結構圖。如果裡面混入「費用申請」或「每週固定例會」，會讓圖表失去工作架構的意義。

**適合取消的例子**：純行政雜務、為了計時才建立的會議任務（若你是被動參與者）

---

#### 顯示於甘特圖

**核心問題：這個任務有時間壓力、需要在時間軸上被追蹤嗎？**

| 情況 | 建議 |
|------|------|
| 任務有明確的截止日，需要對齊整體時程 | ✅ 勾選（**且必須填預估開始/完成日期**） |
| 任務是持續性的、沒有固定截止日 | ❌ 不勾選 |

**為什麼重要？** 甘特圖的作用是一眼看出哪些任務在同一時間競爭你的精力、哪些快到期。一個「永遠在進行的維運支援」出現在甘特圖上沒有意義，還會干擾你判斷真正有截止日的工作。

**注意**：勾選但沒填日期，儲存時會出現錯誤提示。

**適合取消的例子**：固定例會、持續維運、無固定截止日的學習計畫

---

#### 顯示於週報進度表

**核心問題：這個任務的進展需要在定期報告中呈現嗎？**

| 情況 | 建議 |
|------|------|
| 這是主要工作產出，主管或團隊會關心進度 | ✅ 勾選 |
| 這是固定例行活動，不需要每週報告 | ❌ 不勾選 |

**為什麼重要？** 進度表回答「這期我推進了哪些工作？」。如果裡面混入每週都出現、什麼都沒變的任務，報表的訊號會被雜訊淹沒。

但請注意：若一個會議任務有工作產出需要被記錄（如主持例會並產出會議紀錄），則**應該勾選**，因為你每週的產出活動本身就是有意義的進展。

**適合取消的例子**：你只是被動出席但無任何責任的固定會議、費用申請等行政事務

#### 組合範例：六種典型任務的設定

| 任務類型 | 追蹤完成度 | 顯示 WBS | 顯示甘特 | 顯示進度表 |
|---------|-----------|---------|---------|-----------|
| **專案核心任務**（功能開發、文件交付） | ✅ | ✅ | ✅ | ✅ |
| **長期學習研究**（無截止日） | ✅ | ✅ | ❌ | ✅ |
| **固定例會（你是主持人/有產出責任）** | ❌ | ✅ | ❌ | ✅ |
| **固定例會（你只是被動出席）** | ❌ | ❌ | ❌ | ❌ \* |
| **持續維運支援**（On-call） | ❌ | ✅ | ❌ | ✅ |
| **人員招募**（過程無法量化） | ❌ | ✅ | ✅ \*\* | ✅ |

> \* 純被動出席的例會：若四個都不勾，你甚至可以考慮不建任務，直接在排程視圖建立不關聯任務的 Timeslot 即可（選「固定會議」時間分類）。這樣時間仍計入統計，但不佔用任務管理的空間。
>
> \*\* 人員招募若有目標入職日期，可以勾選甘特圖並填入截止日；若時程不確定則取消。

---

### 4.6 任務排序

每個任務列的操作欄提供四個方向鈕：

| 按鈕 | 說明 |
|------|------|
| ↑ 上移 | 在同層兄弟中上移一位 |
| ↓ 下移 | 在同層兄弟中下移一位 |
| ← 提升層級 | 脫離父任務，提升為父任務的同層 |
| → 降低層級 | 移入前一個兄弟任務下，成為其子任務 |

### 4.7 封存

- **單一封存**：在操作欄點擊封存按鈕（僅 DONE/CANCELLED 任務顯示）
- **批次封存**：點擊「封存所有已完成」按鈕，一次封存所有 DONE 和 CANCELLED 的任務
- 封存時，所有子任務會一併封存
- 封存的任務不再出現在任務管理頁面，但其 Timeslot **仍計入**統計報表與週報工時

### 4.8 刪除

點擊刪除鈕後會彈出確認對話框。若任務有子任務，會顯示紅色警示「將一併刪除 X 個子任務」。刪除後可透過 Undo 復原（最多 20 步）。

> **刪除 vs 封存**：任務若有工時紀錄，建議封存而非刪除，以保留統計資料的完整性。

## 5. 工作產出追蹤

工作產出頁面讓你按時間區間查看各任務的產出紀錄，並可直接更新完成度。

### 5.1 控制列

- **統計區間**：選擇開始/結束日期
- **← → 箭頭**：前後跳一週
- **顯示模式**：三種視角切換
  - **按任務**：以任務為主體，列出各任務在此區間的工時與產出，可直接修改完成度
  - **按任務分類**：將同一分類的任務工時加總，適合了解各專案/領域的工時佔比
  - **按時間分類**：依 Timeslot 的時間分類加總，了解各活動類型的工時

### 5.2 資料表格

| 欄位 | 說明 |
|------|------|
| 任務/分類名稱 | 依模式顯示任務名稱或分類名稱 |
| 累計工時 | 該區間內的工時加總 |
| 工作產出 | 縮排子列，顯示產出名稱、類型、連結/摘要 |
| 完成度 | 「按任務」模式可直接在欄位內輸入修改 |

> **提示**：「按時間分類」模式適合用來做週週比較——你這週在「程式開發」vs「議題討論」各花了多少時間？

## 7. 週報生成

週報生成可自動產出 PlantUML WBS 圖、甘特圖，以及 AsciiDoc 格式的進度追蹤表。

> **關於 PlantUML**：本系統可以在頁面上直接渲染圖片（透過 PlantUML 公開伺服器），也可以複製原始碼貼到任何支援 PlantUML 的工具（如 VS Code 的 PlantUML 外掛、Confluence、draw.io）中使用。若公司網路限制外部請求，可以選擇「複製原始碼」並在本地渲染。

### 7.1 報告類型

| 類型 | 區間 | WBS/甘特圖範圍 | 進度表範圍 |
|------|------|----------------|-----------|
| **週報** | 一週 | 當週 | 當週 |
| **雙月盤點** | 兩個月 | 本雙月對（**未來計畫**） | 前一雙月對（**過去成果**） |
| **半年報** | 六個月（12-5月/6-11月） | 當期半年 | 當期半年 |

> **雙月盤點的雙軌設計**：WBS/甘特圖展示「接下來兩個月我要做什麼」（計畫展望），進度表展示「過去兩個月我做了什麼」（成果回顧）。上方會同時顯示兩個期間的日期範圍，避免混淆。

### 7.2 篩選控制

- **WBS 層級篩選**：勾選要顯示的階層（1～5 層）。建議週報只顯示 1-2 層，避免圖太複雜
- **分類排除**：取消勾選不需要呈現在報告中的任務分類
- **甘特圖模式**（週報限定）：週報模式 / 工作盤點模式
- **標示今日**：是否在甘特圖上標示當日
- **顯示原始碼**：切換顯示 PlantUML 原始碼或渲染圖片

### 7.3 WBS 圖

自動生成 PlantUML WBS（Work Breakdown Structure）圖：
- 依任務分類分組
- 以狀態自動著色（完成=綠、取消=黃、暫停=粉、待辦=藍）
- 可複製原始碼或另存為圖片

### 7.4 甘特圖

自動生成 PlantUML 甘特圖：
- 顯示任務的預估時程與完成度百分比
- 依狀態着色（進行中=水藍、暫停=橘、完成=綠、取消=銀）
- 假日和週末以淺藍色標示（在系統設定中維護假日清單）
- DONE/CANCELLED 任務使用**實際日期**（從 Timeslot 推算），反映真實完成時間

### 7.5 進度追蹤表

進度表分為兩張，讓有進展與無進展的任務一目了然：

**有進展任務表**

| 欄位 | 說明 |
|------|------|
| 任務/產出名稱 | 任務標題 + 分類，工作產出以 ↳ 縮排顯示 |
| 預期完成日 | 預估完成日期 |
| 前期 % | 前一期末的完成度快照 |
| 本期 % | 本期的完成度快照（無快照時顯示目前值並加註「(目前)」） |
| Δ 增量 | 本期與前期的差值（↑ 正向進展 / ↓ 退步 / → 持平） |
| SPI | 時程績效指標：`已完成工作量 ÷ 計畫工作量`。SPI≥1.0（綠）表示超前，0.8–1.0（黃）輕微落後，<0.8（紅）嚴重落後 |
| 狀態 | 任務狀態 Chip |

> **SPI（Schedule Performance Index）** 用預估日期和完成度計算你的時程效率。若一個任務預計已過 60% 時程，但完成度只有 40%，SPI 約為 0.67，代表進度落後。SPI 只有在任務有預估日期且追蹤完成度時才有意義。

**無進展任務表**

顯示本期沒有推進的任務，額外有「原因/說明」欄：
- PAUSED 任務顯示暫停原因
- 其他無進展任務顯示「—」（可作為下次審查的提示）

**判斷「有無進展」的規則：**

| 條件 | 判定 |
|------|------|
| 完成度 delta ≠ 0（有快照變化） | 有進展 |
| 本期有 Timeslot 活動紀錄 | 有進展 |
| 本期有週期型產出（effectiveDate 在區間內） | 有進展 |
| 狀態為 DONE | 一律有進展 |
| 狀態為 PAUSED | 一律無進展 |
| 「不追蹤完成度」的任務 | 僅依 Timeslot 與產出活動判斷 |

**快速新增本期產出（⊕ 按鈕）**

進度表每個任務列名稱旁都有一個綠色 ⊕ 按鈕，點擊後開啟小對話框，可直接新增一筆本期產出，歸屬期間自動帶入目前進度表的期間起始日，不需手動填寫。

**AsciiDoc 輸出**

進度表下方提供 AsciiDoc 格式的表格原始碼（含複製按鈕），可直接貼到 AsciiDoc 文件中。

> **什麼是 AsciiDoc？** 一種純文字格式的技術文件語言，類似 Markdown，廣泛用於技術寫作。若你的週報使用 Word 或 Notion，可以忽略這個區塊，只使用 WBS 圖和甘特圖即可。

### 7.6 完成度趨勢圖

進度表任務名稱旁的 📈 按鈕（僅有快照資料時顯示）可開啟趨勢圖 Dialog，以折線圖顯示該任務（及其工作產出）的歷史完成度變化。X 軸為週次，Y 軸為 0–100%，`connectNulls` 模式連接非連續快照。

### 7.7 期間工作成果彙總（雙月盤點/半年報）

進度表下方額外顯示統計彙總區塊：
- 任務狀態分佈（各狀態數量 Chip）
- 各任務分類的工時統計表（從 Timeslot 計算）
- 已完成的工作產出清單（effectiveDate 在期間內的產出）

## 9. 系統設定

### 9.1 資料備份與還原

- **匯出完整資料**：下載包含所有任務、Timeslot、分類、產出類型、假日的 JSON 檔案（檔名含日期，如 `workscope-backup-2026-03-08.json`）
- **還原備份資料**：上傳先前匯出的 JSON 檔，覆蓋所有現有資料（操作前會跳出確認警示）

> ⚠ **定期備份很重要！** 資料僅存在瀏覽器 localStorage，清除瀏覽器資料（如清除 Cookie/快取、重新安裝瀏覽器）會導致全部遺失。建議每週五下班前備份一次。

### 9.2 任務分類管理

新增、編輯、刪除「任務分類」（mainCategories）。

- 修改分類名稱時，所有使用該分類的任務會同步更新
- 刪除分類時，現有任務的分類欄位不會被清除（仍保留舊值，但無法從下拉選單選取）
- 可單獨匯出/匯入分類設定，方便在不同裝置間同步

### 9.3 時間分類管理

新增、編輯、刪除「時間分類」（subCategories），用於 Timeslot 的活動類型標記。

預設分類（建議的程式設計師分類）：
- 固定會議、臨時會議、議題討論
- 思考規劃
- 閱讀學習
- 文件撰寫
- 程式開發、程式碼審查、Debug/問題排查

### 9.4 工作產出類型

管理產出類型定義，每個類型標記為「有形」或「無形」：

| 類型 | 有形/無形 | TaskForm 顯示欄位 |
|------|---------|----------------|
| 實體產出（文件、程式碼、系統） | 有形 | 相關連結（URL/路徑） |
| 決策 | 無形 | 說明/摘要 |
| 知識/研究 | 無形 | 說明/摘要 |
| 流程/規範 | 無形 | 說明/摘要 |

### 9.5 假日/個人休息日

新增或移除特定日期（格式：yyyy-MM-dd）。這些日期會在甘特圖中以淺藍色標示，幫助更準確地呈現可用工作天數。

> **個人排班提示**：若你的休假日與國定假日不同（例如輪班制、彈性假日），可以手動維護這份清單，讓甘特圖更貼近你的實際工作時程。

## 10. 使用情境範例

### 情境一：日常工作時間記錄

> 小明是後端工程師，每天用 WorkScope Planner 記錄工作時間。

1. 早上開啟「排程視圖」，看到今天的時間軸
2. 9:00 開始寫程式，先完成一段後，**點擊 9:00 的空白處**快速新增紀錄
3. 在「快速新增」對話框：選擇任務「使用者登入 API 開發」，時間分類「程式開發」，時間 09:00–11:30，按「確認」
4. 11:30 開會，再點 11:30 空白處，選任務「每週技術週會」，分類「固定會議」，時間 11:30–12:30
5. 發現上午的結束時間打錯了？**單擊那個色塊**，修正後按「儲存」
6. 想看看這個任務的詳細資料？**雙擊色塊**，直接跳到 TaskForm 編輯

### 情境二：建立專案 WBS 結構

> 小華負責管理一個軟體專案，需要建立完整的工作分解結構。

1. 前往「任務管理」，點擊「建立任務」
2. 建立第一階任務「電商平台開發」，勾選「顯示於 WBS」和「顯示於甘特圖」，填入預估日期
3. 在該任務列點擊「建立子任務」按鈕，新增「購物車功能」、「結帳流程」、「商品管理」
4. 在「購物車功能」下再建子任務：「API 設計」、「前端實作」、「整合測試」
5. 每個任務設定預估開始/完成日期、負責人、任務分類
6. 使用 ↑↓ 調整任務順序，← → 調整層級
7. 前往「週報生成」，即可看到完整的 WBS 圖和甘特圖

### 情境三：週報撰寫

> 每週五下午，小明需要撰寫週報。

1. 前往「週報生成」，確認報告類型為「週報」、期間為當週
2. 確認各任務的完成度在本週有更新（到任務管理頁面編輯，或在工作產出頁直接修改）
3. 排除不需要出現在報告的分類（如純行政類）
4. 點擊 WBS 圖的「複製原始碼」或「另存圖片」
5. 點擊甘特圖的「另存圖片」，下載為圖檔插入週報
6. 查看進度追蹤表，確認各任務的進展呈現正確
7. 若有任務出現在「無進展」表，確認是否需要補充說明或補填完成度
8. 點擊「複製 AsciiDoc」，將進度表貼入週報文件

### 情境四：雙月工作盤點

> 每兩個月，需要做一次工作盤點。

1. 前往「週報生成」，切換報告類型為「雙月盤點」
2. 系統自動分為兩個時間軸：
   - WBS/甘特圖顯示**未來兩個月**的工作計畫
   - 進度追蹤表顯示**過去兩個月**的工作成果
3. 頁面上方同時顯示兩個期間的範圍，方便核對
4. 下方的「期間工作成果彙總」自動統計：任務狀態分佈、各分類工時、已完成的產出
5. 將各區塊的內容複製到盤點報告中

### 情境五：固定例會任務的正確設定

> 小明每週主持「技術債討論週會」，每週需要整理會議紀錄和行動項目清單。

**這是「有產出責任」的例會，建議設定如下：**

1. 建立任務「技術債討論週會」，任務分類「內部協作」
2. 設定四個追蹤選項：
   - **追蹤完成度 %**：❌ 不勾選（例會不會「完成」）
   - **顯示於 WBS**：✅ 勾選（這是你的工作職責之一）
   - **顯示於甘特圖**：❌ 不勾選（沒有固定截止日）
   - **顯示於週報進度表**：✅ 勾選（每週有會議紀錄產出需要呈現）
3. 每次開完會，在排程視圖記錄 Timeslot（時間分類：固定會議）
4. 在任務的工作產出區塊，點擊「新增產出」，新增「W10 技術債討論紀錄」，設定歸屬期間為本週
5. 下次開會前，點擊產出旁的 📋「複製為下期」按鈕，自動建立下週的產出條目（歸屬期間自動 +7 天）
6. 在週報進度表中，此任務會出現在「有進展」欄，並顯示本週的會議紀錄產出

> 若你只是被動出席某個會議、沒有任何責任，不需要建立任務。直接在排程視圖建立 Timeslot（不關聯任務，時間分類選「固定會議」）即可，時間仍計入統計。

### 情境六：管理長期無法量化的任務

> 「工程師招募」任務無法定義「完成了幾 %」，但有明確的招募目標日期。

1. 建立任務「後端工程師招募」，任務分類「人員管理」
2. 追蹤完成度：❌、顯示 WBS：✅、顯示甘特圖：✅（填入目標入職日）、顯示進度表：✅
3. 每週的招募活動（電話面試、履歷篩選）記錄為 Timeslot
4. 每個階段的產出（如「錄取 offer 核可」、「背景調查完成」）新增為工作產出
5. 在週報進度表中，系統根據 Timeslot 活動判斷進展，不依賴完成度百分比

### 情境七：任務暫停與追蹤

> 「API 重構」任務因等待外部廠商回覆而暫停。

1. 編輯該任務，狀態改為「已暫停」
2. 系統強制要求填寫暫停原因：「等待廠商 API 規格文件，預計 3/20 前回覆」
3. 在週報進度表中，此任務自動歸入「無進展」表，並顯示暫停原因
4. 廠商回覆後，編輯任務改回「進行中」，繼續更新完成度

### 情境八：資料備份與裝置搬遷

> 要換新電腦，需要遷移 WorkScope Planner 的資料。

1. 在舊電腦：前往「系統設定」→ 點擊「匯出完整資料」→ 下載 JSON 檔
2. 在新電腦：開啟 WorkScope Planner → 前往「系統設定」→ 點擊「還原備份資料」→ 選擇 JSON 檔
3. 所有任務、時間紀錄、分類設定、假日完整還原

> 💡 建議每週五備份一次，並將備份檔存放在雲端（如 Google Drive、iCloud），防止本機遺失。

### 情境九：補填歷史快照

> 上週因為太忙忘記更新完成度，這週的週報「前期 %」欄顯示「—」。

1. 前往「任務管理」，編輯該任務
2. 展開「完成度歷史快照」區塊
3. 在「新增快照」的日期欄選擇「上週任意一天」（系統自動對齊到該週週日）
4. 填入上週實際的完成度數值，點擊「新增」
5. 回到週報生成，前期欄位現在會正確顯示上週的快照值

### 情境十：臨時任務的時間記錄

> 同事突然來找你幫忙排查一個線上 bug，花了 45 分鐘。不想為此建立任務。

1. 在排程視圖點擊空白格，跳出「快速新增」對話框
2. 「任務」欄位留空不選
3. 時間分類選「Debug/問題排查」，說明填「協助 Alice 排查 production 訂單重複 bug」
4. 按「確認」→ 此 Timeslot 在統計中歸為「未分類」，工時計入「Debug/問題排查」時間分類
5. 之後若發現這種臨時協助很頻繁，可以建立一個「技術支援」任務，再回來把這筆 Timeslot 補關聯

## 附錄 C：常見問題

**Q：週報進度表的「前期 %」一直顯示「—」**
A：代表系統找不到前一期的完成度快照。請確認：(1) 任務有勾選「追蹤完成度 %」；(2) 上一期有更新過完成度（或手動補填快照）。

**Q：任務出現在「無進展」表，但我這週確實有在做**
A：系統無法偵測到進展。請確認：(1) 本週有建立 Timeslot 並關聯到該任務；或 (2) 有更新完成度百分比並儲存；或 (3) 有新增本週的工作產出（設定 effectiveDate 在本週）。

**Q：甘特圖上看不到某個任務**
A：請確認任務有勾選「顯示於甘特圖」，且有填入預估開始/完成日期，且任務分類沒有被排除在報告外。

**Q：Undo 按鈕是灰色的**
A：代表目前沒有可復原的操作（歷史紀錄為空），或已經復原到最初始的狀態。

**Q：為什麼刪除封存庫中的任務沒有 Undo？**
A：封存庫中的「永久刪除」是不可逆操作，系統設計上不提供復原。若有重要任務，建議先匯出備份後再刪除。
