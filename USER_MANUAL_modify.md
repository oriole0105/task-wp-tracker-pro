## 1. 快速開始

### 1.1 啟動應用程式

```bash
cd task-time-tracker
npm run dev
```

瀏覽器開啟 `http://localhost:5173` 即可使用。所有資料存在瀏覽器的 `localStorage` 中，無需後端伺服器。

### 1.2 首次使用建議流程

1. **前往「系統設定」** → 新增你的任務分類（如：開發、會議、管理）和時間分類（如：前端、後端、研究）
2. **前往「任務管理」** → 建立你的第一批任務，設定 WBS 階層關係
3. **回到「排程視圖」** → 在時間軸上記錄你每天的工作時間
4. **查看「統計報表」** → 觀察你的工時分配
5. **使用「週報生成」** → 產出 PlantUML 圖表與進度追蹤表

## 2. 全域功能

### 2.1 導覽列

頁面頂部的 AppBar 提供所有頁面的快速導覽：

| 按鈕 | 頁面 | 路徑 |
|------|------|------|
| 任務管理 | WBS 任務清單 | `/tasks` |
| 工作產出 | 產出追蹤 | `/outputs` |
| 排程視圖 | Outlook 風格時間軸 | `/` |
| 統計報表 | 圓餅圖工時統計 | `/reports` |
| 週報生成 | PlantUML 圖表＋進度表 | `/weekly` |
| 封存庫 | 封存任務管理 | `/archive` |
| 系統設定 | 分類/假日/備份 | `/settings` |
| 使用者手冊 | 本使用說明頁 | `/help` |

### 2.2 使用者手冊

導覽列右側的 `?` 按鈕（HelpOutline 圖示）可開啟本使用者手冊頁面（`/help`）。手冊內容依 Markdown 格式渲染，支援完整的章節、表格、程式碼區塊。

### 2.3 復原（Undo）

導覽列右側的 ↩ 按鈕可復原最近的操作（**最多 20 步**）。支援復原的操作包括：新增/編輯/刪除任務、新增/編輯/刪除時間紀錄、封存/還原任務、調整任務排序等。

### 2.4 深色模式

導覽列右側的 🌙/☀ 按鈕切換深色/亮色主題。設定會自動儲存，下次開啟時維持上次的選擇。

## 3. 排程視圖（Dashboard）

排程視圖是你日常記錄工作時間的主要介面，採用 Outlook 行事曆風格，以直覺的視覺化方式呈現每天的時間分配。

### 3.1 檢視控制

頁面上方提供以下控制項：

| 控制項 | 說明 |
|--------|------|
| **日期選擇器** | 點擊選擇要檢視的日期 |
| **← → 箭頭** | 快速切換前/後一天（日模式）或一週（週模式） |
| **檢視模式** | `日`（單天）、`週5`（週一～五）、`週7`（週日～六） |
| **縮放密度** | `精簡`（每小時 60px）、`詳細`（每小時 120px） |
| **色塊顏色** | 以「任務分類」或「時間分類」區分色塊顏色 |
| **顯示說明** | 開關 Timeslot 的滑鼠懸停備註提示 |

### 3.2 時間軸

- 顯示 0:00 到 23:00 的完整 24 小時，開啟時自動捲動到早上 7:00
- 今天的日期欄位有藍色背景高亮
- 紅色水平線標示目前時刻
- 已記錄的 Timeslot 以色塊呈現，顯示任務名稱（優先顯示別名）與時間範圍

### 3.3 新增時間紀錄

**點擊時間軸空白處** → 彈出「快速新增時間紀錄」對話框：

- **任務**（選填）：從未封存的任務清單中選取，也可不選（未分類）
- **時間分類**（選填）：如前端、會議、研究等
- **日期**：自動帶入你點擊的那一天
- **開始/結束時間**：自動對齊到 15 分鐘，結束時間預設加 30 分鐘
- **說明**（選填）：備註文字
- **「建立新任務」捷徑**：若想記錄的任務尚未建立，可直接從此開新任務

### 3.4 時間重疊防護

系統**禁止建立時間重疊的 Timeslot**，確保工時統計不會重複計算：

- **新增時**：若新紀錄的時間範圍與現有 Timeslot 重疊，會顯示錯誤提示（頂部紅色 Snackbar），列出衝突的任務名稱與時間。對話框保持開啟，可直接調整時間後重試。
- **編輯時**：修改時間範圍後，系統會排除自身再檢查是否與其他 Timeslot 重疊。
- **ICS 匯入時**：逐筆檢查，自動跳過與現有紀錄重疊的事件，匯入完成後提示成功與跳過的筆數。

> 💡 進行中的計時（尚未結束、無 endTime）不受重疊檢查影響。

### 3.5 編輯與刪除時間紀錄

**單擊** Timeslot 色塊 → 開啟「編輯時間紀錄」對話框，可修改所有欄位或刪除該筆紀錄。

### 3.6 從 Timeslot 快速編輯任務

**雙擊** Timeslot 色塊 → 直接開啟該任務的 TaskForm 編輯對話框（系統以 250ms 延遲區分單擊與雙擊）。

## 4. 任務管理

### 4.1 WBS 階層結構

任務支援最多 **5 層**的父子階層關係，以 WBS 編號呈現（如 1、1.1、1.1.2）。子任務會以縮排方式顯示在父任務下方。

### 4.2 篩選與搜尋

頁面上方提供多維度篩選：

- **搜尋欄**：全文搜尋任務名稱、別名、說明
- **狀態篩選**：多選，可同時篩選多個狀態
- **分類篩選**：多選，依任務分類篩選
- **標籤篩選**：多選，依自訂標籤篩選
- **日期範圍**：依預估開始/完成日期篩選
- **縮合/展開**：一鍵折疊或展開所有有子任務的任務列

### 4.3 任務狀態

| 狀態 | 說明 | 可封存 |
|------|------|--------|
| 待規劃 (BACKLOG) | 尚未排入執行計畫 | 否 |
| 待處理 (TODO) | 已排入，等待執行 | 否 |
| 進行中 (IN_PROGRESS) | 正在執行 | 否 |
| 已暫停 (PAUSED) | 暫時停止（必須填寫原因） | 否 |
| 已完成 (DONE) | 已完成 | 是 |
| 已取消 (CANCELLED) | 已取消 | 是 |

### 4.4 任務表格

| 欄位 | 說明 |
|------|------|
| No. | WBS 編號，可點擊折疊/展開子任務 |
| 任務名稱 | 標題、別名（灰色小字）、標籤（Chip） |
| 任務分類 | 所屬分類 |
| 預估日期 | 預估開始～完成日期，逾期任務會標紅色 |
| 實際日期 | 從 Timeslot 自動計算（最早開始/最晚結束） |
| 狀態 | 狀態 Chip ＋ 完成度百分比 |
| 累計工時 | 所有關聯 Timeslot 的工時加總 |
| 操作 | 排序 ↑↓、層級調整 ←→、編輯、複製、建子任務、封存、刪除 |

### 4.5 建立/編輯任務（TaskForm）

TaskForm 對話框包含以下欄位群組：

**基本資訊**
- 任務名稱（必填）
- 別名（最多 10 字，排程視圖優先顯示）
- 上層任務（Autocomplete 下拉，支援關鍵字搜尋，依 WBS 編號排序顯示）
- 任務詳細說明
- 標籤（最多 3 個，可搜尋/新增/刪除）

**分類與狀態**
- 任務分類（從系統設定中選取）
- 任務狀態（六種狀態，暫停時須填原因）

**時程**
- 預估開始/完成日期（若勾選顯示於甘特圖則為必填）
- 實際日期（唯讀，自動由 Timeslot 推算）

**人員**
- 任務負責人
- 任務指派人

**追蹤設定**
- 整體完成度 %（0–100，步長 5）— 僅在勾選「追蹤完成度」時顯示
- 三個核取方塊 + 一個三選一選項：
  - **追蹤完成度 %**：取消勾選後，隱藏完成度欄與歷史快照區塊（適用於固定會議、持續型任務）
  - **顯示於 WBS**：控制週報 WBS 圖
  - **顯示於週報進度表**：控制週報進度追蹤表
  - **甘特圖顯示方式**（三選一，預設「進度列」）：
    - **進度列**：顯示為帶完成度百分比的甘特進度條（選取時日期為必填）
    - **章節標題**：渲染為 `-- 任務名稱 --` 分隔行，適合橫跨整個報告週期的大型父任務；子任務各自有自己的甘特設定
    - **不顯示**：完全不出現在甘特圖中

**完成度歷史快照**（僅編輯模式＋追蹤完成度時可見）
- 展開後顯示各週快照清單（可新增、編輯數值、刪除）
- 日期自動對齊至該週週日（weekStart）
- 有快照時自動顯示**折線圖**（任務整體 + 各工作產出各一條線）

**工作產出**
- 可新增多筆產出，每筆設定名稱、類型（有形/無形）、完成度
- 有形產出填寫「相關連結」；無形產出填寫「說明/摘要」
- 可選填「歸屬期間」（週期型產出）
- 每筆產出各自有完成度快照管理

### 4.6 任務排序

每個任務列的操作欄提供四個方向鈕：

| 按鈕 | 說明 |
|------|------|
| ↑ 上移 | 在同層兄弟中上移一位 |
| ↓ 下移 | 在同層兄弟中下移一位 |
| ← 提升層級 | 脫離父任務，提升為父任務的同層 |
| → 降低層級 | 移入前一個兄弟任務下，成為其子任務 |

### 4.7 封存

- **單一封存**：在操作欄點擊封存按鈕（僅 DONE/CANCELLED 任務顯示）
- **批次封存**：點擊「封存所有已完成」按鈕，一次封存所有 DONE 和 CANCELLED 的任務
- 封存時，所有子任務會一併封存
- 封存的任務不再出現在任務管理頁面，但其 Timeslot 仍計入統計

### 4.8 刪除

點擊刪除鈕後會彈出確認對話框。若任務有子任務，會顯示紅色警示「將一併刪除 X 個子任務」。刪除後可透過 Undo 復原。

### 4.9 複製任務

操作欄的複製按鈕（ContentCopy 圖示）可快速複製一個任務：
- 標題自動加上「（副本）」後綴
- 複製欄位：別名、說明、主分類、指派人/報告人、標籤、所有顯示設定、父任務（同層）
- 重設欄位：狀態 → TODO、完成度清空、日期清空、工作產出不帶入
- 支援 Undo 還原

### 4.10 JSON 批次匯入任務

任務管理工具列的「匯入 JSON」按鈕可從 JSON 檔案批次匯入任務：

1. 點擊「匯入 JSON」→ 選擇 `.json` 檔案
2. 彈出確認 Dialog，顯示即將匯入的任務數量與清單預覽
3. 選擇目標父任務（WBS 下拉選單，留空 = 最上層）
4. 點擊「確認匯入」

**JSON 格式**（支援 `{ "tasks": [...] }` 或頂層直接為陣列）：

```json
{
  "tasks": [
    {
      "title": "主任務名稱",
      "description": "任務說明",
      "mainCategory": "Development",
      "labels": ["標籤1", "標籤2"],
      "children": [
        { "title": "子任務名稱" }
      ]
    }
  ]
}
```

可用欄位：`title`（必填）、`aliasTitle`、`description`、`mainCategory`、`assignee`、`reporter`、`labels`、`showInWbs`、`showInReport`、`trackCompleteness`、`ganttDisplayMode`、`status`、`children`（子任務陣列）。

範例檔案：`docs/sample-import.json`

## 6. 統計報表

統計報表以圓餅圖視覺化你的工時分配。

### 6.1 操作

1. 選擇統計區間（日期起訖）
2. 使用箭頭快速切換前/後一天或一週

### 6.2 圓餅圖

並排顯示兩個圓餅圖：

## 7. 週報生成

週報生成是本系統最強大的功能之一，可自動產出 PlantUML 圖表和 AsciiDoc 進度追蹤表。

### 7.1 報告類型

| 類型 | 區間 | WBS/甘特圖範圍 | 進度表範圍 |
|------|------|----------------|-----------|
| **週報** | 一週 | 當週 | 當週 |
| **雙月盤點** | 兩個月 | 本雙月對（**未來計畫**） | 前一雙月對（**過去成果**） |
| **半年報** | 六個月（12-5月/6-11月） | 當期半年 | 當期半年 |

### 7.2 篩選控制

- **WBS 層級篩選**：勾選要顯示的階層（1～5 層）
- **分類排除**：取消勾選不需要的任務分類
- **甘特圖模式**（週報限定）：週報模式 / 工作盤點模式
- **時間刻度**：每日 / 每週 / 每月（切換時自動調整縮放值）
- **依主類別分組**（預設關閉）：開啟後，甘特圖任務依 `mainCategory` 自動加上 `-- 分類名稱 --` 分隔標題
- **標示今日**：是否在甘特圖上標示當日
- **顯示原始碼**：切換顯示 PlantUML 原始碼或渲染圖片

### 7.3 WBS 圖

自動生成 PlantUML WBS（Work Breakdown Structure）圖：
- 依任務分類分組
- 以狀態自動著色（完成=綠、取消=黃、暫停=粉、待辦=藍）
- 可複製原始碼或另存為圖片

### 7.4 甘特圖

自動生成 PlantUML 甘特圖：
- 顯示任務的預估時程
- 標示完成度百分比
- 依狀態着色（進行中=水藍、暫停=橘、完成=綠、取消=銀）
- 假日和週末以淺藍色標示
- DONE/CANCELLED 任務使用實際日期（從 Timeslot 推算）

### 7.5 進度追蹤表

進度表分為兩張：

**有進展任務表**

| 欄位 | 說明 |
|------|------|
| 任務/產出名稱 | 任務標題 + 分類，工作產出以 ↳ 縮排顯示 |
| 預期完成日 | 預估完成日期 |
| 前期 % | 前一期末的完成度快照 |
| 本期 % | 本期的完成度快照（無快照時顯示目前值並加註） |
| Δ 增量 | 本期與前期的差值（↑ 正向/↓ 退步/→ 持平） |
| SPI | 時程績效指標（≥1.0 綠/≥0.8 黃/<0.8 紅） |
| 狀態 | 任務狀態 Chip |
| 說明 | 雙擊可 inline 編輯備註（儲存至 WeeklySnapshot.note） |

**無進展任務表**

顯示本期完成度無變動或暫停中的任務，額外有「原因/說明」欄：
- PAUSED 任務顯示暫停原因（`pauseReason`）
- 其他任務預設顯示「—」
- **雙擊**該欄位可 inline 編輯備註（支援多行，Shift+Enter 換行），儲存至 WeeklySnapshot.note

> **判斷「有無進展」的規則：**
> - 完成度 delta ≠ 0 → 有進展
> - 本期有 Timeslot 活動紀錄 → 有進展
> - 本期有週期型產出（effectiveDate 在區間內）→ 有進展
> - DONE 任務 → 一律有進展
> - PAUSED 任務 → 一律無進展
> - 「不追蹤完成度」的任務 → 僅依 Timeslot 與產出活動判斷

**AsciiDoc 輸出**

進度表下方提供 AsciiDoc 格式的表格原始碼，可直接複製貼到文件中。

### 7.6 完成度趨勢圖

進度表任務名稱旁的 📈 按鈕可開啟趨勢圖 Dialog，顯示該任務（及其工作產出）隨時間的完成度變化折線圖。

### 7.7 期間工作成果彙總（雙月盤點/半年報）

進度表下方額外顯示統計彙總區塊：
- 任務狀態分佈（各狀態數量 Chip）
- 各任務分類的工時統計表
- 已完成的工作產出清單

## 8. 封存庫

### 8.1 封存庫介面

- 搜尋欄：全文搜尋封存任務
- 分類篩選：多選篩選
- 表格：顯示任務名稱（含父任務標示）、狀態、分類、預估完成日、累計工時、封存時間

### 8.2 操作

| 操作 | 說明 |
|------|------|
| **還原** | 將任務（含所有子任務）移回任務管理，恢復正常狀態 |
| **永久刪除** | 完全刪除任務與子任務（有確認對話框，**無法 Undo 復原**） |

## 9. 系統設定

### 9.1 資料備份與還原

- **匯出完整資料**：下載包含所有任務、Timeslot、分類、產出類型、假日的 JSON 檔案（檔名含日期，如 `workscope-backup-2026-03-08.json`）
- **還原備份資料**：上傳先前匯出的 JSON 檔，覆蓋所有現有資料（操作前會跳出確認警示）

> ⚠ 建議定期備份！資料儲存在瀏覽器 localStorage 中，清除瀏覽器資料會導致遺失。

### 9.2 任務分類管理

新增、編輯、刪除「任務分類」（mainCategories）。修改分類名稱時，所有使用該分類的任務會同步更新。

可單獨匯出/匯入分類設定，方便在不同裝置間同步。

### 9.3 時間分類管理

新增、編輯、刪除「時間分類」（subCategories）。用於 Timeslot 的活動類型標記。

### 9.4 工作產出類型

管理產出類型（如：實體產出、決策、知識/研究、流程/規範）。每個類型標記為「有形」或「無形」：
- **有形產出**：TaskForm 中顯示「相關連結」欄位
- **無形產出**：TaskForm 中顯示「說明/摘要」欄位

### 9.5 假日/個人休息日

新增或移除特定日期。這些日期會在甘特圖中以淺藍色標示，幫助更準確地呈現工作時程。

## 10. 使用情境範例

### 情境一：日常工作時間記錄

> 小明是前端工程師，每天用 WorkScope Planner 記錄工作時間。

1. 早上開啟「排程視圖」，看到今天的時間軸
2. 9:00 開始寫程式，先完成一段後，**點擊 9:00 的空白處**快速新增紀錄
3. 在「快速新增」對話框中：選擇任務「使用者登入頁面開發」，時間分類「前端」，時間 09:00-11:30，按「確認」
4. 11:30 開會，再點 11:30 空白處，選任務「每週團隊會議」，分類「會議」，時間 11:30-12:00
5. 發現上午的紀錄結束時間打錯了？**單擊那個色塊**，在編輯對話框修正，按「儲存」
6. 想看看這個任務的詳細資料？**雙擊色塊**，直接跳到 TaskForm 編輯

### 情境二：建立專案 WBS 結構

> 小華負責管理一個軟體專案，需要建立完整的工作分解結構。

1. 前往「任務管理」，點擊「建立任務」
2. 建立第一階任務「前端開發」，勾選「顯示於 WBS」和「顯示於甘特圖」
3. 在該任務列點擊「建立子任務」按鈕，新增「登入模組」、「儀表板」、「報表模組」
4. 在「登入模組」下再建子任務：「UI 設計」、「API 串接」、「測試」
5. 每個任務設定預估開始/完成日期、負責人、任務分類
6. 使用 ↑↓ 調整任務順序，← → 調整層級
7. 前往「週報生成」，即可看到完整的 WBS 圖和甘特圖

### 情境三：週報撰寫

> 每週五下午，小明需要撰寫週報。

1. 前往「週報生成」，確認報告類型為「週報」、期間為當週
2. 檢查 WBS 圖是否涵蓋所有工作項目（可調整層級篩選）
3. 排除不需要的分類（如固定會議類）
4. 點擊 WBS 圖的「複製原始碼」，貼到週報文件中
5. 點擊甘特圖的「另存圖片」，下載為圖檔插入週報
6. 查看進度追蹤表，確認各任務的完成度變化
7. 若有任務出現在「無進展」表，檢查是否需要補充說明
8. 點擊「複製 AsciiDoc」，將進度表貼入 AsciiDoc 週報範本

### 情境四：雙月工作盤點

> 每兩個月，團隊需要做一次工作盤點。

1. 前往「週報生成」，切換報告類型為「雙月盤點」
2. 系統自動分為兩個時間軸：
   - WBS/甘特圖顯示**未來兩個月**的工作計畫
   - 進度追蹤表顯示**過去兩個月**的工作成果
3. 頁面上方顯示兩個期間的範圍，方便核對
4. 下方的「期間工作成果彙總」自動統計：任務狀態分佈、各分類工時、已完成的產出
5. 將各區塊的內容複製到盤點報告中

### 情境五：管理長期持續型任務

> 「每週部門例會」是一個長期進行的任務，不適合用完成度百分比來追蹤進度。

1. 前往「任務管理」，編輯「每週部門例會」
2. 取消勾選「追蹤完成度 %」→ 完成度欄位和歷史快照區塊自動隱藏
3. 每週的會議產出用「工作產出」欄位記錄，設定歸屬期間為該週日期
4. 在週報進度表中，系統會依 Timeslot 活動和產出記錄判斷進展，而非完成度

適用場景：固定會議、人員招募、持續性維運工作等無法量化完成度的任務。

### 情境六：任務暫停與追蹤

> 「API 重構」任務因等待外部廠商回覆而暫停。

1. 編輯該任務，狀態改為「已暫停」
2. 系統強制要求填寫暫停原因：「等待廠商 API 規格文件」
3. 在週報進度表中，此任務自動歸入「無進展」表，並顯示暫停原因
4. 廠商回覆後，編輯任務改回「進行中」，繼續更新完成度

### 情境七：資料備份與裝置搬遷

> 小明要換新電腦，需要遷移 WorkScope Planner 的資料。

1. 在舊電腦：前往「系統設定」→ 點擊「匯出完整資料」→ 下載 JSON 檔
2. 在新電腦：開啟 WorkScope Planner → 前往「系統設定」→ 點擊「還原備份資料」→ 選擇 JSON 檔
3. 所有任務、時間紀錄、分類設定、假日完整還原

> 💡 建議養成每週備份一次的習慣，防止瀏覽器資料意外遺失。

### 情境八：檢視完成度趨勢

> 想了解某個長期任務過去幾週的進度變化。

**方式一：從 TaskForm 查看**
1. 編輯該任務，展開「完成度歷史快照」區塊
2. 底部自動顯示折線圖：X 軸為週次、Y 軸為 0–100%
3. 若任務有多筆工作產出且各有快照，會以不同顏色線條同時呈現

**方式二：從週報生成查看**
1. 前往「週報生成」，在進度追蹤表找到該任務
2. 點擊任務名稱旁的 📈 圖示
3. 彈出趨勢圖 Dialog，完整顯示歷史走勢

### 情境九：從無任務的時間紀錄開始

> 有些工作是臨時性的，不需要建任務，只想記錄花了多少時間。

1. 在排程視圖點擊空白格，跳出「快速新增」對話框
2. 「任務」欄位留空不選
3. 只填時間分類（如「臨時支援」）和說明（如「協助 QA 重現 bug」）
4. 按「確認」→ 此 Timeslot 在統計中歸為「未分類」
5. 之後若想歸類，可單擊色塊編輯，補選任務
